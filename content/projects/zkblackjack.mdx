## SYSTEM OVERVIEW

zkBlackjack is an on-chain singleplayer and multiplayer blackjack system where each round outcome is verified using zero-knowledge proofs rather than trusted backend execution.

Game correctness is enforced cryptographically. Players prove valid outcomes without exposing hidden game state such as cards or intermediate decisions.

The project investigates how interactive real-time applications can operate securely on public blockchain infrastructure.

## ARCHITECTURE

**Game Contract Layer**

Smart contracts manage wagers, round lifecycle, and payout settlement. Funds remain locked on-chain until proof verification completes.

**ZK Verification Layer**

Circom circuits generate proofs confirming that blackjack rules were followed correctly and that the declared winner is valid.

Proofs are verified on-chain before settlement.

**Realtime Coordination Layer**

Socket-based synchronization enables multiplayer sessions while maintaining deterministic blockchain settlement.

**Frontend Interface**

Next.js client handles gameplay interaction and local proof generation using snarkjs.

## ENGINEERING CHALLENGES

- **Private game state**

  Blackjack requires hidden card information. Designed circuits capable of verifying legality of moves without revealing player hands.

- **Proof latency**

  Browser-side proof generation optimized to maintain playable round timing.

- **Multiplayer synchronization**

  Coordinating realtime sessions while final authority remains on-chain.

- **Gas-aware verification**

  Reduced verifier complexity to maintain feasible Ethereum execution costs.

## SYSTEM SIGNIFICANCE

zkBlackjack serves as an experimental model for:

- private multiplayer applications
- zk-verified interaction systems
- trustless game execution
- cryptographic state validation

Beyond gaming, the architecture applies directly to private auctions, prediction markets, and agent coordination systems.
